# 会议室占用冲突排查与调度台

一个本地可运行的 Web 控制台应用，帮助行政人员高效管理会议室预约，自动检测时间冲突，支持改期、取消、房间锁定、黑名单管理等功能，并提供完整的操作日志和数据导出能力。

## 功能特性

### 核心功能
- **预约管理**：导入预约、查看预约列表、筛选条件（按房间、状态、组织者、日期）
- **冲突检测**：自动识别同一会议室的时间重叠，高亮显示冲突记录
- **冲突处理**：支持改期、取消预约记录
- **房间管理**：锁定/解锁会议室（仅管理员）
- **黑名单管理**：管理组织者黑名单，黑名单用户无法创建新预约
- **备注处理**：添加和编辑预约备注
- **历史日志**：记录所有操作历史（创建、更新、取消、改期、锁房、解锁、黑名单操作）
- **数据导出**：导出预约数据、房间数据、黑名单数据和操作日志

### 用户角色
- **管理员**：完整权限，可执行所有操作
- **普通用户**：仅查看权限，无权锁房

### 数据持久化
- 使用 LocalStorage 存储所有数据，重启后数据继续保留

## 技术栈

- React 18 + TypeScript
- Vite 构建工具
- TailwindCSS 3 样式框架
- Zustand 状态管理
- Lucide React 图标库

## 使用说明

### 成功链路验证

#### 1. 冲突检测
- 打开控制台首页，系统已预置样例数据
- 会议室 A 有两条冲突预约（张三 09:00-10:00，李四 09:30-10:30）
- 冲突记录会以红色高亮显示，并标注冲突图标

#### 2. 改期操作
- 在控制台首页，找到冲突的预约记录
- 点击改期图标（铅笔图标），展开改期表单
- 输入新的时间（如 10:30-11:30），点击"确认改期"
- 改期成功后，预约状态变为"已改期"，冲突自动解除

#### 3. 取消操作
- 在控制台首页，找到待取消的预约记录
- 点击取消图标（垃圾桶图标），预约状态变为"已取消"
- 已取消的预约不再参与冲突检测

#### 4. 导出数据
- 导航到"数据导出"页面
- 点击各导出按钮，可分别导出预约数据、房间数据、黑名单数据和操作日志
- 导出格式为 CSV，可直接用 Excel 打开

#### 5. 房间锁定（管理员模式）
- 确保当前为管理员模式（左下角显示"管理员"）
- 导航到"房间管理"页面
- 点击房间卡片上的"锁定"按钮
- 房间状态变为"已锁定"，无法再创建新预约

#### 6. 黑名单管理
- 导航到"黑名单"页面
- 点击"添加黑名单"按钮，输入组织者姓名和原因
- 被加入黑名单的组织者无法创建新预约

### 冲突与权限失败验证

#### 1. 重叠预约互相覆盖（失败路径）
- 尝试将一条预约改期到与另一条预约重叠的时间
- 系统会提示"改期失败：新时间与其他预约冲突"
- 改期操作不会生效

#### 2. 普通用户无权锁房（失败路径）
- 切换为普通用户模式（左下角"切换为普通用户"）
- 导航到"房间管理"页面
- 尝试点击"锁定"按钮，系统提示"普通用户无权锁定房间"
- 锁定操作不会生效

#### 3. 改期后取消保留冲突链
- 对一条冲突预约进行改期操作
- 改期成功后再取消该预约
- 在操作日志中可查看完整的操作记录链

## 数据存储

所有数据存储在浏览器 LocalStorage 中：

| Key | 说明 |
|-----|------|
| `room-scheduler-reservations` | 预约列表 |
| `room-scheduler-rooms` | 房间列表 |
| `room-scheduler-blacklist` | 黑名单 |
| `room-scheduler-logs` | 操作日志 |
| `room-scheduler-current-user` | 当前用户 |

刷新页面或重启浏览器后，所有数据将自动恢复。
